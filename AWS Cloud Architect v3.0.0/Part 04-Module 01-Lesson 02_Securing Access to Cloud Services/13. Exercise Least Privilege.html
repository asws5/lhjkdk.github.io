least Privilege Access Exercise Instructions

Exercise: Least Privilege Access

You have been tasked with setting up a new IAM policy that will be used by an application.

We will use the Role that you had created in the previous exercise in this lesson (Exercise - Leveraging IAM roles): cand-c3-iam-role

  1. Create a new S3 bucket for this exercise. Call it cand-c3-s3-app1-yourname or similar. Note down the name of the bucket.
  2. Create a second bucket to test denial of access. Call it cand-c3-s3-offlimits-yourname or similar.
  3. Upload a test file to both buckets.
  4. Go to the IAM service in the console and create a new IAM policy. Call the IAM policy cand-c3-iam-policy-least-priv.
  5. The IAM policy will need to be able to do the following:
  • Upload objects to an S3 bucket
  • Download objects from an S3 bucket
  • List objects that are in the S3 bucket
  • Only be able to perform these options on the new S3 bucket you have created.
  • List S3 buckets
  • Deny the ability to download files (s3:getObject) from any other S3 buckets that are not the S3 bucket you created.
  • Helpful Examples: Example 1 and Example 2
  1. In the IAM console go to the role that you had created in the previous exercise: cand-c3-iam-role
  2. Assign the new policy you created to the role and remove the managed policy that had previously been assigned to the role.
  3. Test the policy by assuming the role and attempting to download the file that you had uploaded from both buckets. You should be denied downloading the file from the offlimit bucket.
  4. Go back to your admin user.
  5. Add a managed policy to the cand-c3-iam-role which allows full S3 access.
  6. Assume the role and test the policy again. The original restriction should still be in place due to the deny you had added in the custom IAM policy.

You have successfully fine-tuned a policy and achieved a level of least privilege access for a role!
