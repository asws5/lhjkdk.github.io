Lambda Functions Lesson Recap

Key Points About Lambda Events and Functions

  • Invoking Lambda functions means triggering them, or kicking them off.

  • AWS automatically provisions CPU and network resources based on the amount of memory you select to run your functions

  • Lambda functions can be written in a variety of coding languages: Node.js, Python, Go, Java, C# and more

  • The serverless code you write and configure for one cloud platform can only be used on that platform so using AWS Lambda to run your applications ties you to AWS

Key Points about Lambda@Edge

  • Lambda@Edge allows you to execute lambda functions in edge locations that are geographically closer to the user.

  • Running code closer to users improves user satisfaction

  • An example of a function that can be executed in an edge location is rewriting the response URL based on device type- serving a mobile user a mobile optimized page

How to Create a Lambda Function

  1. Login to the AWS Console
  2. Browse to the IAM service and create and name a role with the
    AWSLambdaSQSQueueExecutionRole policy
  3. Browse to the AWS Lambda Service management console
  4. Select Create Lambda, then select Author From Scratch
  5. Name your Lambda function and select the Python 3.8 runtime
  6. Select the IAM role that was created in step 2 and create the function.
  7. Browse to the AWS Simple Queue Service Management console
  8. Create and name an SQS queue, configure trigger for Lambda function
  9. Return to the Lambda management page and configure an SQS trigger with the SQS queue created in step 7.
  10. Edit the Python code for the Lambda function to print a message when it is triggered
  11. On the SQS management page, add a message to the queue
  12. Browse to the CloudWatch management page, and view the logs. Locate the log entry for the Lambda function and observe the result.

Congratulations! You have created and triggered a Lambda function!
