My Solution

For this exercise, I created a table with names and information about the neighbors I interact with when I am walking around the neighborhood. I capture some fun and practical data points about each neighbor, but, as you see, not every neighbor has relevant

Here are the steps I took to create my table and add items

  • I created a DynamoDB table named udacity_neighborhood with neighbor as the partition key
  • I added neighbors using Create item and string values such as animal, car, grocery, and garden for each neighbor
  • I modified my udacity_neighborhood table in the table view using the pencil edit icon
  • I used the search feature to search c and found cooper and C and found Charlie, demonstrating that search in DynamoDB is case sensitive
