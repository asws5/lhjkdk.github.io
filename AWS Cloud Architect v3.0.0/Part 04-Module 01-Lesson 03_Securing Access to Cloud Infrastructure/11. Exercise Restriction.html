11. Exercise: Restriction

Restrict Access to Cloud Resources Exercise Instructions

Configure Security Groups and ACLs

In this exercise, you will choose a VPC and public subnet in your AWS account to setup a new NACL and security group and test both.

  1. Setup a new public subnet in your VPC for this exercise.

  2. Create a new Network ACL for the VPC.

  3. Observe the inbound and outbound rules for the newly created NACL.

  4. Associate the new NACL with the new public subnet you have created.

  5. Create a new security group in the same VPC. Do not add any rules at this time. Note down the security group name.

  6. Determine your public IP address (Hint: Google what's my IP).

  7. Launch a linux EC2 instance into the public subnet you have created. Assign it a public IP address and the new security group that was created.

  8. Attempt to ping the EC2 instance on its public IP address.

  9. Attempt to ssh into the EC2 instance.

  10. Update the NACL. Add an inbound rule, number 100, allowing TCP traffic. All ports (0) all IPs (0.0.0.0/0).

  11. Add an outbound rule, number 100, allowing all traffic, all ports, all IPs.

  12. Although your ACL now allows SSH, attempt to SSH and ping the instance. Neither should work since the security group has not been updated.

  13. Update the security group to allow SSH connections from your public IP address. Also allow ICMP traffic from your public IP address.

  14. Again attempt the SSH connection, the connection should succeed (you don’t need to login). Attempt the ping, it will still fail since we haven’t allowed ICMP traffic on the NACL.

  15. Update the NACL. Add a rule to allow ICMP traffic from all IPs.

  16. Try the ping again. This time it should succeed.

  17. Update the NACL. Remove the outbound rule that allows all traffic. Now the outbound portion of the NACL should be a simple deny all.

  18. Attempt the ping and SSH again. Note that they no longer work since the return traffic has been blocked by the NACL.

  19. Re-add the NACL allowing all outbound traffic. Verify that connectivity is restored.

  20. Add an explicit deny in the NACL. Deny your IP inbound TCP connectivity on port 22.

  21. Attempt the SSH connection again. Notice that it no longer works.
