Techniques to provide secure access to servers Exercise Instructions

Secure Access With Systems Manager

In this exercise, you will use AWS Systems Manager to SSH into an EC2 instance in your AWS account.

  1. Create an Instance Profile Role.
  • Refer to the AWS documentation for setting up an instance profile role for AWS Systems Manager. Skip straight to Task 2 in this AWS documentation.
  • You may name the role cand-c2-l2-instance-role.
  1. Launch An EC2 Instance.
  • When choosing the AMI, use Quick Start and select the Amazon Linux 2 AMI.
  • For the IAM role, assign the name of the role you just created: cand-c3-l2-instance-role.
  • The rest of the options can be left default or of your own choosing.
  • Please refer to the AWS documentation for more information on launching an EC2 instance.
  1. Connect to your EC2 instance. Follow the AWS documentation for Systems Manager to connect to the EC2 instance you have just created.
