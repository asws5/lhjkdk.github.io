Purchasing Options

When optimizing your computing usage to reduce your monthly spend, one of the first places you want to count your costs is in your EC2 instances, and one of the best resources here is using reserved instances.

Reserved and Spot instances

Reserved instances offer a savings of almost 70% in some cases. In exchange for an upfront commitment of 1 or 3 years, AWS offers significant savings, which increases a bit depending on how you choose to pay. Paying in advance will save you even more.

Spot instances can potentially save customers 90% of AWS on-demand EC2 instance pricing, but there are some significant drawbacks. AWS can terminate a spot instance that is running below the current spot call price at any time after providing you with a 2-minute warning.

Think Like a Cloud Architect

QUESTION:

Reflect

How would you architect this EC2 scenario with a combination of On Demand. Reserved, and Spot instances for maximum cost effectiveness?

  • Running an application on an instance that needs to capture metrics for 7 days straight
  • A batch process that runs for 3 hours every night
  • A website that has been selling inflatable kayaks for 15 years
ANSWER:

Thanks for your response. For an application that only needs to run for a week, it's acceptable to use an on-demand instance. It is best to use spot instances for batch processes that only need to run for a few hours and if AWS needs to stop your running instance and restart it later, there won’t be any lasting effects. If you are running a website that has longevity and you project that it will be running long into the future, it would be most cost effective to purchase reserved instances.

Remember

Key points

  • Reserved instances are a 1 or 3 year commitment and multiple payment options also save money,

  • Spot instances can be terminated by AWS at any time with a 2 minute notification that you might not see

  • On demand instances are the most expensive option and should only be used in short-term or urgent situations.
