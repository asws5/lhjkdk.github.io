05. Solution: Pricing by Region

Pricing Calculator Detailed Steps

My Solution Using the AWS Pricing Calculator

Here is how I completed this exercise:

For EC2

  1. Select Create Estimate, search for and select Amazon EC2, then select Configure.
  2. Select the desired AWS Region from the Region dropdown.
  3. Select Quick Estimate and Linux Operating System
  4. Choose Search instances by name and search for c5.xlarge.
  5. Add 8 instances to the estimate.
  6. Under Pricing strategy, select On-Demand Instances
  7. Leave the defaults under Elastic Block Storage.
  8. At the bottom of the page, click Add to my estimate, and on the next page, select Add service.

For Application Load Balancer

  1. Search for and select Elastic Load Balancing, then select Configure
  2. Choose one Application Load Balancer.
  3. Under Average connection duration, type 1.
  4. Under Average number of requests per second per ALB, type 10.
  5. Select Add to my estimate, then select Add service.

For S3

  1. Search S3 (Amazon Simple Storage Service), then select Configure.
  2. Toggle S3 Standard ON , and the remaining options OFF.
    16 Under S3 Standard storage type 4 , then select TB from the dropdown.
  3. Select Add to my estimate, then select Add service.

For AWS RDS

  1. Search RDS to find Amazon Aurora (MySQL Compatible), then select Configure.
  2. Select Aurora Serverless (MySQL Compatible).
  3. Add 2 Aurora Capacity Units per hour and 100GB of Database Storage.
  4. Select Add to my estimate

Further Research

Key Points

As we can see in this exercise, costs vary by region. And even if the region is in the USA, it doesn’t mean it is less expensive. N California is more expensive than Northern Virginia and Mumbai.

There are many reasons for the price difference. Supply and demand, local government regulations and taxes, the cost of doing business such as land, construction and electricity.

Application performance and user satisfaction are the goals of a cloud architect. You will need to decide where to host your application, balancing costs with your performance goals. In some cases you will need to host your application in multiple regions in order to meet your obligations to your users. A strong understanding of how region affects cost will be useful as you advance in your careers.
