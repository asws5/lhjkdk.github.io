15. Exercise: Security Event

Security Event Sources Exercise Instructions

Viewing Activity With CloudTrail

In this exercise, you will make some changes to your environment and verify that your activity can be audited using CloudTrail.

1. Generate some activity

  • Log out of your AWS account and log back in
  • Create a new bucket in your AWS account. Ensure that you select default encryption.
  • Once the bucket is created, go back and remove the default encryption setting.

2. Search CloudTrail Event History

  • After a few minutes, go to the CloudTrail service console and go to Event History.
  • Search for the login event by choosing "Event Name" in the first filter box and pasting the text ConsoleLogin into the second filter box. You should not only see your login that you just performed but all logins in the last 90 days.
  • Expand the event to see more details. Click View Event.

3. Expected Results

  • You should be able to see the IP address you logged in from, the user name, time, and other valuable details.
  • Search again, this time on the filter choose "Resource Name" and in the second box put the S3 bucket name that you just created.

4. Expected Results

  • You should see all activity related to creating the bucket and removing the encryption setting.

Continue to explore the CloudTrail history in your account to get a better feel for the types of management and API calls being made.
