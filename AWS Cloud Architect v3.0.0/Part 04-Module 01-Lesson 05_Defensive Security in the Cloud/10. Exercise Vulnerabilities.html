10. Exercise: Vulnerabilities

Identify Vulnerabilities in AWS Exercise Instructions

Find Misconfigurations Using AWS Config

In this exercise, you will get a feel for how AWS Config can help identify vulnerabilities in your AWS account.

  1. Login to your AWS Account and Turn on Config Recording
  • Go to the AWS Config service page.
  • Follow the instructions here to set up config recording
  • Check the Choose Specific Resources option to see all available AWS resource types that can be monitored with AWS Config. Select S3 Buckets for this exercise.
  • Please note down which S3 bucket you have used to store config logs.
  1. Set Up a Rule to Monitor S3 Bucket Encryption
  • Go to Rules and add a rule. On this page, you can browse through the many preconfigured rules that are available in AWS config.
  • In the search box, put
    "S3-bucket-server-side-encryption-enabled" and select the rule.
  • On the next page, leave all the defaults, and click save.
  1. Check Rule Compliance
  • Scroll through the list of rules until you find the rule that was added: s3-bucket-server-side-encryption-enabled
  • You can see if the rule is in compliance or not. Click the rule name to see more details.
  • You can view details related to specific resources that are in compliance or out of compliance.
  • Choose a bucket that is not in compliance. If there or no buckets in the list go ahead and create a new S3 bucket for this exercise that does not have default encryption enabled.
  • If you created a new bucket, verify that it has been evaluated and is now showing in the list.
  • Go to the bucket of choice and update the default encryption setting on the bucket.
  • Wait roughly 5 minutes for the change to trigger a rule check and check the config rule again.
  • Change the filter in the rule to show in compliance resources only. You should see the bucket listed.
  • Click the bucket name to see further details. Click on the "configuration timeline".
  • You can see details of when the s3 bucket was changed. Try to use this page to see what specifically changed in terms of the S3 configuration.
  1. Cleanup
  • Delete the rule and S3 bucket that you have created for this exercise.
